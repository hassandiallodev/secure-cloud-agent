# Definition of Done (DoD)

## DoD d’une séance (obligatoire)
Une séance est considérée terminée si :

- [ ] Objectif de la séance atteint (résultat visible)
- [ ] Code / config committés sur Git (pas de travail “non sauvegardé”)
- [ ] Push sur GitHub effectué
- [ ] CI GitHub Actions verte (workflow CI ✅)
- [ ] README / docs mis à jour si nécessaire (au moins 3 lignes si nouvelle étape)
- [ ] Notes rapides de séance ajoutées (ce que j’ai fait + ce que je ferai ensuite)

## Règles de qualité
- Branch `main` doit rester stable (pas de commit cassé)
- Les commandes doivent fonctionner en CLI (pas seulement dans l’IDE)
- Ne jamais utiliser l’utilisateur root AWS pour travailler (sécurité)
